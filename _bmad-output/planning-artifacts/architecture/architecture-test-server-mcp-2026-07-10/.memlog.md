- (decision) Paradigm: layered architecture with Presentation/Application/Domain/Infrastructure layers. Binds all components. Prevents architectural drift where layers become entangled. Rule: dependencies must point inward toward domain layer only.
- (decision) AD-1: Programmatic interface first. Binds all capabilities. Prevents UI from becoming primary delivery mechanism. Rule: MCP server is authoritative; HTTP UI is purely convenience.
- (decision) AD-2: Test runner abstraction. Binds all test execution capabilities. Prevents Vitest-specific code leaking into app/domain layers. Rule: Each runner has dedicated adapter implementing common interface; app layer depends on abstraction only.
- (decision) AD-3: Coverage mapping ownership. Binds coverage intelligence features. Prevents multiple components claiming ownership of coverage data. Rule: Coverage Tracker is sole owner; all queries go through it.
- (decision) AD-4: Immutable results. Binds all result reporting. Prevents inconsistent views during long runs. Rule: Test results are immutable snapshots; new results replace old ones atomically.
- (decision) AD-5: Dependency direction. Binds all layers. Prevents circular dependencies and architectural entropy. Rule: Dependencies point inward toward domain; domain knows nothing about outer layers.
- (decision) Stack: TypeScript 5.x, Node.js 20+, @modelcontextprotocol/sdk, Vitest, pnpm 8+. Verified versions on npm registry as of July 2026.
- (decision) Structural seed: src/{mcp/,http/,application/,domain/,infrastructure/,shared/}, tests/, package.json, mcp.json. Clear separation of concerns with layer-based directories.
- (decision) Conventions: PascalCase for classes/interfaces, camelCase for functions/variables, plural for collections. UUID v4 for IDs, ISO 8601 for timestamps, standardized error envelope.
- (deferred) Full CI/CD integration patterns — Post-MVP phase; current focus is local development experience.
- (deferred) Multi-project workspace support — Complex enough to warrant separate epic after single-project works.
- (deferred) Advanced caching strategies (distributed cache, cache invalidation heuristics) — MVP can use simple in-memory cache.
- (event) spine finalized
- (decision) AD-1a: Dry run mode support. Binds all test orchestration capabilities. Prevents executing unwanted tests during agent iteration. Rule: Server must support dry run mode where tests can be queued and evaluated without execution.
- (decision) PARADIGM SUPERSEDED: not a single-process layered app — a singleton DAEMON (control plane) that spawns one isolated WORKER SUBPROCESS per project (data plane) communicating over IPC. Process isolation is the top-level invariant; the layered inward-dependency rule (AD-5) now applies only within daemon code.
- (decision) AD-6: Transport = Streamable HTTP (StreamableHTTPServerTransport, stateful sessions), one daemon per system enforced by lockfile + known port. Binds all client access. Prevents multiple competing daemons / port races. Rule: bind 127.0.0.1 only, mandatory Host+Origin validation, per-daemon bearer token; optional stdio single-project mode.
- (decision) AD-7: Project-local execution. Binds all test execution. Prevents daemon's own deps contaminating results / version skew across projects. Rule: each run happens in a forked per-project worker with cwd=project root that resolves vitest/node from the PROJECT's node_modules; daemon never imports a project's Vitest.
- (decision) AD-8: State topology. Binds all persistence. Prevents black-box central state / project pollution. Rule: per-project state (coverage map, history) in git-ignored <git-root>/.test-mcp/; daemon-global registry+lockfile central in ~/.test-mcp, never inside a project; projectId=hash(abs path), pinnable; every persisted JSON carries schemaVersion.
- (decision) AD-9: Coverage build method (extends AD-3 ownership). Binds coverage map construction. Prevents setup-file pollution making every source look globally depended-on. Rule: build source->test map single-pass from runtime V8 coverage, subtract the setup-file baseline, and mark any unmeasurable test always-run. Spike-validated on a large frontend app.
- (decision) AD-10: Recall-prioritised selection. Binds test-selection logic. Prevents missed failures from over-aggressive skipping. Rule: on ANY uncertainty (unknown/new file, setup-baseline module, unmeasurable test) fall back to full suite.
- (version) Stack refined: @modelcontextprotocol/sdk v1 (McpServer + StreamableHTTPServerTransport), Node 20+ child_process.fork workers, Vitest via vitest/node (target repo pins 4.1.9). Drops mcp.json single-project assumption.
