- (decision) Intent: Build MCP server for intelligent test running. Binds all capabilities. Prevents building wrong thing. Rule: Focus on Vitest first, extend later.
- (decision) Capability C1: Run tests programmatically via configured runners. Binds all test execution features. Prevents missing core functionality. Rule: Server must expose run_tests tool.
- (decision) Capability C2: Intelligent incremental runs based on coverage mapping. Binds coverage intelligence features. Prevents unnecessary test re-runs. Rule: System must track coverage and determine dependent tests on file change.
- (decision) Capability C3: Real-time result streaming. Binds all result reporting. Prevents stale views during long runs. Rule: Results stream as they complete, not batched at end.
- (decision) Capability C4: Minimal failure-focused output for AI consumption. Binds output formatting. Prevents verbose noise. Rule: Summary includes failures only; details available on request.
- (decision) Boundary: In-scope includes Vitest (first), watch mode, coverage tracking, HTTP UI. Out-of-scope: multi-project workspace, distributed caching, CI/CD integration (post-MVP). Binds scope definition. Prevents feature creep. Rule: MVP focuses on single-project Vitest experience.
- (decision) Success metrics: <5 min full suite, <30s incremental, 95%+ coverage accuracy, <1s UI latency, zero false negatives. Binds quality goals. Prevents settling for subpar performance. Rule: All metrics measured on slow hardware baseline.
- (deferred) Multi-project workspace support — Complex enough for separate epic after single-project works.
- (event) spec finalized
- (decision) Success criteria aligned with brief: <15s incremental, 99%+ precision, 95%+ coverage mapping, zero false negatives, 80%+ adoption. Binds quality goals. Prevents settling for subpar performance.
- (decision) User profile split: Primary = AI agents (programmatic access), Secondary = Human developers (web UI). Binds all features. Prevents conflating two different user needs.
- (decision) Job statement: AI agents orchestrate intelligent test runs to minimize wait time while ensuring reliability. Binds all capabilities. Prevents building wrong solution.
- (decision) Capability C1: Dry run mode - queue tests without executing. Binds AI agent workflow features. Prevents executing unwanted tests. Rule: Server must support dry run mode for evaluation before commitment.
- (event) PRD updated per PRFAQ findings
- (decision) REDESIGN: product is a single on-system daemon (Streamable HTTP) managing MULTIPLE registered projects; AI registers a project then drives tests via MCP, never invoking vitest directly. Supersedes earlier 'multi-project out-of-scope' boundary — multi-project registration is now CORE Phase 1.
- (decision) Capability CAP-7: Runtime project registration (register_project/list_projects/unregister_project) via thin test-mcp CLI that auto-boots the singleton daemon. Success: an unregistered project dir with a vitest/vite config can be registered and then run via MCP by projectId.
- (decision) Capability CAP-8: Execute tests via each project's OWN Vitest in an isolated per-project worker subprocess (cwd=project root, vitest/node resolved from project node_modules). Success: two projects on different Vitest versions run without daemon contamination.
- (decision) Capability C3 REFRAMED (supersedes real-time streaming): MCP tools/call is request/response; Phase 1 uses progress notifications + pollable get_test_status; genuine real-time push (SSE/WebSocket) is Phase 2 UI only. Success: agent can poll status idle/running/complete/error and receive progress during a run.
- (decision) Capability CAP-9: Coverage-based reverse map (source->test-file) built SINGLE-PASS from runtime V8 coverage, with setup-baseline subtraction and always-run fallback for unmeasurable tests. Spike-validated on a large frontend app. Success: after setup-baseline subtraction, a source change selects ~6% (unit)/~18% (integration) of suite, not the whole suite.
- (decision) Capability C1 dry run refined: plan/commit via planId — dryRun returns a TestPlan (planId, files, reasoning, expiresAt); run_tests({planId}) executes exactly that; expired planId => re-plan.
- (constraint) Per-project state (coverage map + history) persists in git-ignored <git-root>/.test-mcp/ (repo-local, visible, not a black box); daemon-global registry/lockfile lives centrally (~/.test-mcp), never inside a project. projectId defaults to hash of abs path, pinnable in config.json.
- (constraint) Security: daemon binds 127.0.0.1 only, mandatory Host/Origin validation, per-daemon bearer token (CLI-managed). Uses @modelcontextprotocol/sdk v1 (McpServer + StreamableHTTPServerTransport); vitest/node programmatic API; pin Vitest version (target repo 4.1.9).
- (constraint) Correctness: recall prioritised over precision — whenever selection is uncertain (unknown file, setup-baseline module, unmeasurable test) run the FULL suite. Drops the absolute 'zero false negatives' claim in favour of conservative fallback.
- (note) Non-goals: Jest/pytest (future), Human UI (Phase 2), priority scoring + health monitoring (Phase 2), fixture/setup-time tracking + ordering-dependency detection + parallel resource-contention quotas (deferred, research-grade), cross-platform beyond macOS (Phase 2).

- (decision) POSITIONING REFRAME (PRFAQ 2026-07-10 competitive research): coverage-based selection is now TABLE-STAKES, not the differentiator — testpick (single-pass V8 coverage selection), vitest-agent (plugin+MCP+SQLite), vitest-affected, and proposed native vitest --stale already ship it. Differentiator = delivery architecture: single multi-project always-on daemon + project-local Vitest version isolation + transparent repo-local .test-mcp state + setup-baseline subtraction as first-class correctness. Supersedes any implication that C9 is the unique differentiator.

So we PORT/VENDOR testpick's MIT V8 snapshot-diff attribution algorithm into our own worker (keep our daemon/worker/state model, add setup-baseline subtraction ourselves), retaining testpick's MIT copyright+license notice. Not CLI-wrap, not full-scratch.
